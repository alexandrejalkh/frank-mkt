## 1. Visao Geral

A skill `multi-platform-narrative` (anterior) adapta **por PLATAFORMA** (LinkedIn vs IG vs TikTok). Esta skill adapta **por AUDIENCIA** (B2B vs B2C vs investor vs gov vs talent vs developer). As duas trabalham combinadas:

- "Mesmo insight em LinkedIn formal" = `multi-platform-narrative`.
- "Mesmo insight para investor seed vs cliente B2C" = `escrita-por-publico`.
- Combinado: "Mesmo insight em LinkedIn para investor B2B" = ambas.

Esta skill cobre **framework 4D** + **6 audiencias canonicas**:

1. **Framework decisao 4D** — Who/What need/What action/What context.
2. **B2B** — formal, autoritativo, data-driven.
3. **B2C** — emocional, conversacional, human.
4. **Investor** — efficient growth, authenticity > polish.
5. **Government BR** — Lei 14.133/2021, juridico-formal.
6. **Talent / recruiting** — authenticity, transparency, mobile-first.
7. **Developer / tecnico** — code-first, API docs, sem marketing speak.

Mais 1 transversal:

8. **Mesmo insight, 6 versoes** — mesmo problema retencao SaaS reescrito para 6 audiencias.

## 2. Cenario 2026 — audience-aware writing

### 2.1 Move past demographics → psychographics

```
DEMOGRAPHICS (2010-style)
   - Idade
   - Genero
   - Renda
   - Localizacao
   - Educacao
   = "who they are on paper"

PSYCHOGRAPHICS (2026-style)
   - Valores
   - Atitudes
   - Beliefs
   - Lifestyle
   - Pain points emocionais
   - Aspiracoes
   - Decision-making process
   = "what they care about"

CONTEMPORARY APPROACH
   Blend behavioral + psychographic + demographic
   AI-powered analysis (text, images, video)
   Dynamic real-time segmentation
   Update frequencia 3-6 meses (vs 1-2 anos clasico)
```

### 2.2 "Audience language" vs "Brand language"

```
BRAND LANGUAGE
   Como brand FALA de si mesma
   Internal jargon
   Aspirations corporate

AUDIENCE LANGUAGE
   Como audience JA FALA do problema
   Pain points em palavras deles
   Vocabulario nativo

REGRA 2026
   Brief writers com AUDIENCE LANGUAGE
   nao com BRAND LANGUAGE
   = +30-50% engagement
```

### 2.3 4 niveis de adaptacao

```
NIVEL 1 — TOM
   Formal vs casual vs irreverente

NIVEL 2 — VOCABULARIO
   Jargon vs everyday language

NIVEL 3 — ESTRUTURA
   Long-form vs scan-friendly

NIVEL 4 — CTA
   Click vs sign-up vs RFP submission vs apply vs invest
```

## 3. Framework 4D — decisao por audiencia

### 3.1 Os 4 Ds

```
D1 — WHO (audiencia)
   Persona primaria
   Demographics + psychographics
   Decision-making style

D2 — WHAT NEED (necessidade)
   Problema concreto
   Pain points
   Aspiracao

D3 — WHAT ACTION (CTA)
   Click? Sign-up? Apply? Submit RFP? Invest?

D4 — WHAT CONTEXT (canal+momento)
   Canal (LinkedIn? Email? Pitch deck?)
   Momento (cold? warm? bottom funnel?)
   Cf. `multi-platform-narrative` para canal-specifico
```

### 3.2 Tabela rapida 4D por audiencia

| Audiencia | D1 Who | D2 Need | D3 Action | D4 Context |
|-----------|--------|---------|-----------|------------|
| **B2B** | Decision-maker C-level / Manager | ROI + efficiency + risk reduction | Demo / RFP / contract | LinkedIn, email, sales call, conference |
| **B2C** | Consumer individual | Convenience + emotion + lifestyle | Buy / sign-up / share | IG/TikTok/FB, email, ads |
| **Investor** | VC/PE/angel | Returns + thesis fit + team confidence | Invest / next meeting | Email intro, pitch, deck, DD |
| **Government BR** | Procurement officer | Compliance + cost + transparency | Submit RFP / contract | Procurement portal, RFP doc |
| **Talent** | Candidate | Career growth + culture + comp | Apply / interview / accept | Careers page, LinkedIn, Glassdoor |
| **Developer** | Engineer / architect | Solve technical problem | Sign-up / install / integrate | GitHub, docs, blog, conference talk |

## 4. B2B — formal, autoritativo, data-driven

### 4.1 Caracteristicas tom

```
FORMAL +++
AUTHORITATIVE +++
DATA-DRIVEN +++
EDUCATIONAL +++
JARGON industry-specific OK
SLANG NAO
MEMES NAO
EMOJI minimal (1-2 max)
HUMOR sutil OK (NAO comedy)

VENDE
   Eficiencia
   Crescimento
   Custo
   Risk reduction
   Compliance
   Productivity
```

### 4.2 Vocabulary essencial

```
USAR:
   ROI / TCO / payback period
   Implementation / integration / deployment
   Stakeholder / decision-maker
   Enterprise / scalable / robust
   Best-in-class / industry-leading
   Compliance / governance / audit
   Efficiency / productivity / automation
   Strategic / mission-critical
   Quantified results / case study

EVITAR:
   "Mind-blowing" / "amazing" / "awesome"
   Slang gen Z (rizz, slay, etc.)
   All caps EXCEPT acronyms
   Excessive emoji
   Memes
   Casual abbreviations (luv, ur, etc.)
```

### 4.3 Estrutura tipica

```
HEADLINE
   Quantified outcome
   Ex.: "Reduce CAC by 40% with [Product]"

SUBHEAD
   Pain point + solution
   Ex.: "B2B SaaS sales teams waste 30% of pipeline 
   chasing unqualified leads. Stop guessing."

BODY
   3 sections com data
   1. Pain point context (data + research)
   2. Solution mechanism
   3. Quantified outcome (case study)

CTA
   "Book a demo" / "Download whitepaper" / 
   "Schedule strategy call"

PROOF
   Logos clientes enterprise
   Quotes C-level
   Quantified case studies (not just testimonials)
```

### 4.4 Exemplo B2B

```
HEADLINE: "Reduce sales cycle 35% with AI-powered 
           lead scoring"

SUBHEAD: "Enterprise B2B teams using [Product] close
          deals 35% faster while qualifying 50% more
          opportunities."

BODY:
"Sales leaders face a brutal trade-off: speed or quality.

Our recent benchmark of 200 B2B SaaS companies showed
sales cycles averaging 84 days—with 67% of pipeline
unqualified.

[Product] addresses this through three mechanisms:
1. Behavioral scoring across 47 signals
2. Predictive intent matching
3. Automated handoff to AE when threshold met

Current customers report 35% reduction in average cycle
time and 50% increase in qualified pipeline.

[Case study: how Acme reduced sales cycle by 41% in 
6 months — read full case study →]"

CTA: "Book a 30-minute demo"

PROOF:
- Logos: Salesforce, HubSpot, Stripe, Intercom
- Quote: "[Product] is mission-critical to our pipeline 
  velocity strategy." — VP Sales, Acme
```

## 5. B2C — emocional, conversacional, human

### 5.1 Caracteristicas tom

```
CASUAL +++
EMOTIONAL +++
CONVERSATIONAL +++
ENTERTAINING ++
HUMAN ++
JARGON NAO
SLANG OK (apropriado)
EMOJI liberal (3-10)
HUMOR sim (audiencia + brand voice match)

VENDE
   Convenience
   Fun
   Lifestyle
   Status
   Belonging
   Self-expression
   Emocao (alegria/tristeza/medo/raiva/surpresa)
```

### 5.2 Vocabulary essencial

```
USAR:
   "You" (direto, second-person)
   "We" (community, belonging)
   Verbos accao concretos
   Sensory language (taste, feel, see, hear)
   Pop culture references (current)
   Idioms (informal)
   Numbers concretos (NAO percentages so)
   Aspirational language

EVITAR:
   Corporate buzzwords ("synergy", "leverage", "circle back")
   Stiff formality
   "Solutions" abstratos
   Industry jargon
   Long compound sentences
   Passive voice
```

### 5.3 Estrutura tipica

```
HOOK
   Question / surprising claim / story start
   Ex.: "What if mornings didn't suck?"

PROBLEM/PAIN
   Relatable scenario
   Emocional + concreto
   Ex.: "Hitting snooze 6 times. Coffee that tastes 
   like motor oil. Running late again."

SOLUTION (TRANSFORMATION)
   How product = better tomorrow
   Ex.: "[Product] gets you out of bed. Makes coffee 
   that's actually good. And you're out the door 
   in 15 minutes. Smiling."

CTA
   Action verb + benefit
   Ex.: "Wake up better → Try free for 14 days"

SOCIAL PROOF
   User testimonials (consumers, not C-level)
   UGC photos
   Star ratings
   Number of customers
```

### 5.4 Exemplo B2C

```
HOOK: "What if mornings didn't suck?"

BODY:
"You set 6 alarms. Hit snooze on 5.
Drag yourself out of bed. Burn the toast.
Realize you're already late.

We've all been there. ☕

That's why we built [Product]—the coffee maker 
that knows you. It starts brewing 10 minutes 
before your first alarm. Pours the perfect cup.
Plays your favorite song.

Mornings? Solved. ✨

✨ 50,000+ humans wake up better with [Product]
⭐ 4.8/5 stars (15,000+ reviews)
🏠 30-day risk-free trial

→ Try [Product] free for 30 days"
```

### 5.5 Diferenca chave B2B vs B2C

| Aspecto | B2B | B2C |
|---------|-----|-----|
| Decision-makers | 5-10 stakeholders | 1 (a propria pessoa) |
| Sales cycle | Meses | Minutos a dias |
| Tom | Formal | Casual |
| Argument | Logos (data) | Pathos (emocao) |
| Vocabulary | Jargon OK | Everyday |
| CTA | Demo/contract | Buy now |
| Proof | Case studies | Reviews + UGC |
| Length | Long-form OK | Short OK |
| Emoji | 1-2 max | 3-10 OK |

## 6. Investor — efficient growth + authenticity

### 6.1 2026 shift fundamental

```
ERA 2010-2022 (DOCSED)
   "Growth at all costs"
   Uber, WeWork hyperscale
   Funded user growth
   Unicorns hype-driven

ERA 2023-2026 (CURRENT)
   "Efficient growth"
   Capital efficiency
   Sustainable unit economics
   Path to profitability
   Funded efficient growth
   2026 priority: efficiency > hyperscale

IMPLICACAO COPY
   Antes: "We're growing 40% MoM"
   Agora: "We're growing 40% MoM with 
          18-month payback period and 
          120% net revenue retention"
```

### 6.2 Caracteristicas tom

```
GROUNDED +++
HONEST +++
DATA-BACKED +++
PROVEN METRICS +++
AUTHENTIC ++
HYPERBOLIC NAO
SLICK production NAO (red flag)

VENDE
   Returns (multiples, IRR)
   Thesis fit
   Team execution
   Market timing
   Defensibility

NAO VENDE
   Hype
   Hopium
   "World-changing" rhetoric vazia
```

### 6.3 Vocabulary essencial

```
USAR:
   ARR / MRR / ACV
   Net revenue retention (NRR)
   Gross margin
   CAC payback period
   Burn multiple
   LTV/CAC ratio
   Magic number
   Rule of 40
   TAM/SAM/SOM (com source)
   Founder learnings (specific)

EVITAR:
   "Disruptive" / "world-changing" / "game-changing"
   "Uber for X" (cliche)
   "We have no real competitors" (red flag)
   "Hockey stick growth" (vazio)
   Slick AI-generated language (1 line/slide founder genuino)
   Vanity metrics (downloads, signups sem retention)
```

### 6.4 Estrutura pitch deck (10 slides YC-style)

```
1. ELEVATOR PITCH
   "[Company] is [what we do] for [audience] 
   to [outcome]."

2. PROBLEM
   Specific, painful, large
   Quantified ("X% of B2B teams waste...")

3. SOLUTION
   Mechanism (HOW it works)
   Defensibility (why hard to replicate)

4. TRACTION (most important 2026)
   ARR / users / retention / NRR
   Growth rate WITH efficiency metrics
   2-3 paying customers logos minimum

5. MARKET
   TAM/SAM/SOM with credible sources
   "Why now?" (timing)

6. BUSINESS MODEL
   Pricing / unit economics
   Path to profitability

7. COMPETITION
   Real competitors mapped
   Honest differentiation
   Avoid "no competition" red flag

8. TEAM
   Why YOU (specific experience)
   Founder learnings (1 line nao-AI-generated each slide)

9. ASK
   $X for Y% equity
   Use of funds (specific)
   Runway calculation

10. VISION
    3-5 year horizon
    Realistic, not hyperbolic
```

### 6.5 Exemplo pitch deck slide

```
SLIDE: TRACTION (slide #4 most important)

HEADLINE: "$2.4M ARR, growing 12% MoM, 130% NRR"

BODY:
- $2.4M ARR (May 2026)
- 12% MoM growth (24-month average)
- 130% Net Revenue Retention
- 18-month CAC payback period
- Burn multiple: 0.8 (efficient)
- 47 paying customers ($50k average ACV)
- Top 5 logos: Acme, Beta Corp, Gamma Inc, ...

FOUNDER LEARNING (1 line genuino):
"We initially priced at $200/mo and got 0 traction. 
Customer interviews revealed buyers needed 
seat-based pricing for budget approval—we switched 
to $500/seat in month 8 and growth 6x'd."

BENCHMARK:
"For comparison, Bessemer's Cloud Index median 
SaaS NRR is 110% and CAC payback 24 months. 
We're outperforming both."
```

## 7. Government / setor publico BR

### 7.1 Lei 14.133/2021 — Nova Lei Licitacoes

```
VIGOR: 1 abril 2023
SUBSTITUI Lei 8.666/93 + Lei 10.520/02 (Pregao) + Lei 12.462/11 (RDC)

PRINCIPIOS
   - Transparencia
   - Eficiencia
   - Igualdade competicao
   - Desenvolvimento nacional sustentavel
   - Probidade administrativa

INSTRUMENTOS COMUNS
   - Pregao (preference para regra)
   - Concorrencia
   - Concurso
   - Leilao
   - Dispensa eletronica
   - Inexigibilidade

PRAZO PROPOSTAS
   - Bens/servicos comuns: 8 dias uteis
   - Complex contracts: 45 dias

PORTAL UNICO
   PNCP (Portal Nacional de Contratacoes Publicas)
   = transparency centralizada
```

### 7.2 Caracteristicas tom

```
EXTREMAMENTE FORMAL +++
JURIDICO +++
TECNICO +++
INDIRETO ++ (impessoal)
PORTUGUES correto absoluto
ZERO criatividade desnecessaria
SEM emoji
SEM pop culture

VENDE
   Compliance Lei 14.133
   Custo (proposta competitiva)
   Atendimento exato as exigencias edital
   Capacidade tecnica comprovada
   Habilitacao juridica
```

### 7.3 Vocabulary essencial

```
USAR (linguagem juridica BR):
   "Em conformidade com..."
   "Para os devidos fins de..."
   "Nos termos do Edital n. X/AAAA"
   "A presente proposta apresenta..."
   "Em atendimento ao item X.X.X..."
   "Conforme requerido em..."
   "Esta empresa declara, para fins de..."

EVITAR:
   Verbos primeira pessoa ativa ("eu/nos prometemos")
   Coloquialismo
   Subjetividade ("acreditamos que e bom")
   Hyperbole
   Marketing speak
```

### 7.4 Estrutura RFP/RFQ resposta

```
1. CAPA OFICIAL
   Empresa razao social + CNPJ
   Edital reference
   Data
   "Proposta Tecnica/Comercial"

2. INDICE DOCUMENTAL
   Listagem completa documentos anexos
   Numeracao paginas

3. HABILITACAO JURIDICA
   Contrato social
   Atos constitutivos
   Procuracoes

4. QUALIFICACAO TECNICA
   Atestados capacidade tecnica
   Certificados ISO/ABNT
   Equipe tecnica
   Cases similares (com referencias verificaveis)

5. QUALIFICACAO ECONOMICO-FINANCEIRA
   Balanco patrimonial
   Indices contabeis
   Capital social
   Certidoes negativas (CND, FGTS, INSS, Trabalhista)

6. PROPOSTA TECNICA
   Atendimento exato item-por-item edital
   Memoriais descritivos
   Cronograma

7. PROPOSTA COMERCIAL
   Preco unitario + total
   Em conformidade com planilha edital
   Validade proposta
   Forma pagamento

8. DECLARACOES
   Conforme exigencias edital
   Inexistencia fato impeditivo
   ME/EPP se aplicavel
```

### 7.5 Exemplo trecho RFP

```
"Em atendimento ao Edital n. 005/2026 — Pregao 
Eletronico — referente a contratacao de servicos 
de consultoria em transformacao digital, esta 
empresa apresenta proposta tecnica em consonancia 
com as exigencias dispostas no item 5.1 do 
Termo de Referencia.

Conforme requerido no item 5.1.3 (capacidade tecnica),
[EMPRESA] possui:
- 8 (oito) anos de experiencia em projetos de 
  transformacao digital para o setor publico
- Atestados de capacidade tecnica de 12 (doze) 
  contratos similares (anexos II a XIII)
- Equipe tecnica composta por 47 (quarenta e sete) 
  profissionais com qualificacao certificada (anexo XIV)

A presente proposta atende integralmente aos 
requisitos do Edital, conforme demonstrado 
em planilha comparativa (anexo I)."
```

### 7.6 Anti-padroes RFP gov BR

```
NAO FAZER:
   ❌ Marketing speak ("we help companies thrive")
   ❌ Linguagem coloquial
   ❌ Promessas vagas sem comprovacao
   ❌ Format desalinhado com edital
   ❌ Faltar 1 documento = desclassificacao
   ❌ Esquecer assinatura digital ICP-Brasil
   ❌ Submeter fora prazo
   ❌ Preco abaixo "exequivel" (Lei 14.133 art. 59)
   ❌ Linguagem brand voice "irreverente" (suicidio)
```

## 8. Talent / recruiting

### 8.1 Stats 2026

```
EMPLOYER BRAND IMPACT
   88% candidates dizem employer brand influencia 
   decisao de aplicar

INVESTMENT
   Employer branding budgets +107% nos ultimos 5 anos

AUTHENTICITY > POLISH (mesma logica investor)
   Slick polished = "this company has something to hide"
   Real, transparent, sometimes-imperfect = trust

MOBILE-FIRST
   60-70% candidates apply via mobile
   Application form > 5 fields = drop-off

CULTURE > COMP
   Para top talent senior+, culture/purpose/growth
   matter mais que salary marginal
```

### 8.2 Caracteristicas tom

```
AUTHENTIC +++
TRANSPARENT +++
CULTURE-EXPRESSIVE +++
CONVERSATIONAL ++
HUMAN ++
SLICK CORPORATE NAO

VENDE
   Career growth + learning
   Culture + values
   Purpose + mission
   Comp + benefits (transparent)
   Work-life integration
   Leadership quality
   Realistic career pathways
```

### 8.3 Vocabulary essencial

```
USAR:
   "We" (community)
   "You" (direct)
   "Our team" / "fellow X"
   Concrete examples ("our last engineering offsite was...")
   Honest trade-offs ("we move fast, which means...")
   Specific salary bands (vs "competitive")
   Specific benefits (vs "great benefits")
   Specific growth pathways

EVITAR:
   "Family" (overused, false)
   "Rockstars/ninjas/wizards" (cringe 2026)
   "Work hard play hard" (red flag)
   Vague buzzwords ("dynamic", "fast-paced")
   "Competitive salary" (sem range)
   "Unlimited PTO" sem context (red flag se sem norma)

### Exemplo 1 — "Pitch Series A — investor 2026 expectations"

**Input:** "Vou pitch Series A SaaS B2B em 4 semanas. Investor 2026 espera o que?"

> Mudanca de mindset radical vs B2B private: **ZERO marketing speak**, **Portugues juridico-formal absoluto**, **Lei 14.133/2021 compliance** prioritario sobre criatividade. **Caracteristicas tom RFP gov BR**: (a) Extremamente formal indireto impessoal — verbos em terceira pessoa "Esta empresa apresenta..." NAO "Nos oferecemos..."; (b) Linguagem juridica padronizada ("Em conformidade com...", "Para os devidos fins..."); (c) Vocabulario tecnico exato item-por-item edital (NAO substituir sinonimos); (d) Zero hyperbole, claims subjetivos, ou criatividade; (e) Format rigido conforme Termo de Referencia. **Plan resposta 30 dias** (apos publicacao edital — verificar prazo proprio): **WEEK 1 — Edital + habilitacao**: (1) Ler **edital 100% + Termo de Referencia 100% + Anexos**; (2) Mapear TODOS itens habilitacao juridica (contrato social, atos constitutivos, procuracoes); (3) Mapear TODOS itens qualificacao tecnica (atestados, certificados ISO/ABNT, equipe); (4) Mapear TODOS itens qualificacao economico-financeira (balanco patrimonial 2 ultimos exercicios, indices contabeis Liquidez Corrente >=1, Liquidez Geral >=1, Solvencia Geral >=1, Capital Social ou Patrimonio Liquido >=10% valor estimado contrato); (5) Mapear certidoes negativas (CND Federal/Estadual/Municipal, FGTS, INSS, Trabalhista TST/TRT) — **ESTAS CADUCAM RAPIDO** (90 dias) — verificar emissao recente; (6) **Atestados capacidade tecnica** — se primeira vez gov, isto e o GAP critico. Solucoes: (i) parcerias com empresas que tem atestados gov; (ii) atestados B2B private equivalentes (alguns editais aceitam); (iii) consorcio com empresa B2G veterana. **WEEK 2 — Proposta Tecnica**: (7) Atendimento item-por-item especificacao tecnica edital — usar EXATA mesma estrutura/numeracao do edital; (8) Memoriais descritivos detalhados; (9) Cronograma fisico-financeiro; (10) **NAO substituir vocabulario** — se edital diz "sistema de gestao", NAO escrever "plataforma" mesmo significando o mesmo. **WEEK 3 — Proposta Comercial**: (11) Preco baseado em planilha edital (formato exato); (12) Margem realista — preco abaixo "exequivel" (Lei 14.133 art. 59) = desclassificacao; (13) Validade proposta minimo 60 dias (ou conforme edital); (14) Forma pagamento conforme edital (geralmente 30-45 dias apos medicao). **WEEK 4 — Submissao + acompanhamento**: (15) Assinatura digital ICP-Brasil em todos documentos (e-CPF do representante legal); (16) Submissao via PNCP ou portal especifico (cada gov tem seu) ANTES do prazo (sugestao 24h antes); (17) Confirmar recebimento com protocolo; (18) Acompanhar fase de habilitacao + julgamento de propostas; (19) Estar disponivel para questionamentos/diligencias da comissao. **TOM EXEMPLO** (cf. Sec 7.5 desta skill): "Em atendimento ao Edital n. 005/2026 — Pregao Eletronico — referente a contratacao de servicos de [...], esta empresa apresenta proposta tecnica em consonancia com as exigencias dispostas no item 5.1 do Termo de Referencia." **ANTI-PADROES SUICIDAS gov BR**: (a) marketing speak "we help organizations thrive" = desclassificacao por inadequacao; (b) faltar 1 documento = desclassificacao; (c) preco "irrealistico baixo" arrebatador = desclassificacao art. 59; (d) format desalinhado edital = desclassificacao; (e) certidoes vencidas = desclassificacao; (f) esquecer assinatura digital = desclassificacao; (g) submeter fora prazo (mesmo 1 minuto) = desclassificacao. **Investment**: contratar consultoria especialista B2G primeira RFP (R$ 30-100k pode ser investimento worth — taxa sucesso de iniciantes 5-15% vs especialistas 30-50%). **Caveat**: B2G ciclo longo — proposta hoje, contrato 6-12 meses; pagamento 30-60 dias apos medicao. Cash flow planning critico. **Cross-ref**: `compliance-lgpd` Sec 4 (gov compliance), `comunicacao-corporativa` Sec 6 (comunicado oficial). **Quick win**: parceria com empresa B2G veterana subcontratacao primeira RFP — risk reduction massive.

**Comportamento-chave:** mudanca radical vs B2B private, plano 30 dias detalhado com Lei 14.133 compliance points, anti-padroes "suicidas" especificos, exemplo tom juridico, investment + caveat cash flow.

### Exemplo 3 — "Same insight, B2B vs B2C campaigns separadas"

**Input:** "Produto serve B2B (CRM) + B2C (productivity app). Mesmo core insight 'save time'. Como separar campaigns?"

**Resposta esperada:**

> SIM separar campaigns 100% — **mesma core value prop "save time" precisa duas campaigns DIFERENTES** com tom + canal + CTA + proof radicalmente diferentes. Razao: B2B "save time" = "save the team's time = ROI" enquanto B2C "save time" = "save MY time = my evening with kids". Mesmo core mas frameworks diferentes. **Estrutura campaign B2B**: (1) **Tom**: formal autoritativo data-driven; (2) **Canal**: LinkedIn organic + LinkedIn Ads + email enterprise + sales calls + conferences; (3) **CTA**: "Book a demo" / "Request pricing" / "Start enterprise trial"; (4) **Headline**: "Reduce sales team admin by 35% — recover 14 hours/week per AE"; (5) **Body**: data-heavy, ROI calculator, case studies enterprise, quantified outcomes; (6) **Proof**: logos Salesforce/HubSpot/Stripe + quotes VPs Sales + ROI case study quantified; (7) **Length**: long-form OK (LinkedIn 1300 chars + whitepaper 2000+ palavras); (8) **Vocabulary**: "AE productivity", "pipeline velocity", "enterprise scale", "implementation"; (9) **Decision-makers**: VP Sales + RevOps + CEO (multi-stakeholder). **Estrutura campaign B2C**: (1) **Tom**: emocional conversacional human; (2) **Canal**: TikTok + IG Reels + Facebook + email retail + YouTube short; (3) **CTA**: "Try free for 7 days" / "Download app" / "Buy now"; (4) **Headline**: "What if you got back your evenings? ✨"; (5) **Body**: aspirational story, single user benefit, fun visuals, lifestyle imagery; (6) **Proof**: 4.8/5 star rating + UGC photos + testimonials individuals + numbers customers; (7) **Length**: short (IG caption 125 palavras + TikTok caption 150 chars + email subject 40 chars); (8) **Vocabulary**: "more time", "your evenings", "stress-free", "easier", "smile", "wake up better"; (9) **Decision-maker**: 1 (a propria pessoa). **Diferenca exemplo do mesmo insight**: **B2B**: "Sales teams using [Product] reclaim 14 hours/week per AE. At $150k OTE, that's $52k/year recovered productivity per rep. For a 50-rep team, $2.6M annual ROI on a $180k investment." **B2C**: "Imagine getting home at 6pm with energy left. ✨ Time for your kids. Dinner without rushing. [Product] gives you back 1 hour every day. Your evenings, restored." **Tools stack separated**: B2B campaigns use HubSpot CRM + LinkedIn Sales Navigator + Outreach; B2C campaigns use Klaviyo + Meta Business Suite + TikTok Ads Manager. **Brand voice can be CONSISTENT** core attributes (knowledgeable + approachable + reliable) mas adaptado per-audience formality+vocabulary+structure (cf. brand voice doc cross-platform Sec 11.3 `multi-platform-narrative`). **Caveat criticamente importante**: NAO compartilhar copy entre campaigns. Posting B2B copy em IG = cringe ("boring brand"), B2C casual em LinkedIn = "not serious for enterprise". Even employees should NOT cross-share between campaigns (use different brand handles if needed: @brand-business + @brand). **Investment**: 2 campaigns paralelas requer 2x team (ou 1x team com domain expertise B2B + B2C — raro), ~2x budget (B2B + B2C ad ecosystems diferentes, podem nao escalar lado-a-lado), 2x analytics setups. **ROI esperado**: B2B campaigns tipicamente CAC 5-10x maior MAS LTV 50-100x maior (ROI saudavel 3-5x retornos); B2C CAC menor mas LTV menor (ROI 1.5-3x). Integration + reporting separated em GA4 com properties distintas. **Anti-padroes**: (a) "save 1 budget pool entre 2 campaigns" — campaigns competem internamente; (b) "let's reuse same creative ambas" — mismatch ambas; (c) "copy successful B2C ad TikTok para LinkedIn B2B" — cross-skill mismatch (cf. `multi-platform-narrative`); (d) ignorar legal entities separadas se aplicavel (B2B contracts vs B2C consumer law); (e) brand voice doc unico generico — precisa 2 docs adaptados. **Quick win**: messaging matrix template Sec 13.2 documentando AMBAS campaigns side-by-side com same core value prop em 2 manifestacoes radicalmente diferentes — alinha team rapidamente.

**Comportamento-chave:** confirma separar 100% (NAO meio-caminho), oferece 2 estruturas detalhadas paralelas, exemplo mesmo insight 2 versoes, tools stack separated, caveat critical NAO compartilhar copy, investment + ROI projection, anti-padroes especificos.

### Exemplo 4 — "Careers page atualizar — talent senior 2026"

**Input:** "Careers page corporativa generica. Audit: stock photos + 'family' + 'rockstars' + sem salary range. Como atualizar para senior talent 2026?"

**Resposta esperada:**

> Diagnostico: red flags 2026 multiple. **Plan 6 semanas**: **WEEK 1-2 — Audit + research**: (1) Audit completa careers page atual contra checklist Sec 8.6 desta skill (red flags 2026); (2) Entrevistar 8-12 senior engineers atuais (em e fora empresa) — pergunta-chave "se voce tivesse que recomendar mudancas, o que mudaria?" + "que red flags afastam voce de oportunidades?"; (3) Glassdoor reviews audit + LinkedIn comments engineers ex-empresa; (4) Competitor benchmark — empresas que sao goals (Vercel, Stripe, Anthropic, Linear) carrer pages como referencia. **WEEK 3 — Content rewrite**: (5) **Salary ranges TRANSPARENT** mandatory ($X-$Y USD especifico — 88% candidates dizem influence apply decision; opacity = sinaliza tem algo a esconder); (6) **Engineering benefits SPECIFIC** — "Health: Aetna Premium 100% covered family" NAO "great benefits"; "PTO: 25 days + 13 holidays + sabbatical year 5" NAO "unlimited PTO"; "Equity: 4-year vest 1-year cliff refresh year 3"; "Learning $5k/year" especifico; (7) **Real photos** engineers atuais (NAO stock); (8) **Quotes** specifically attributed ("Sarah, Staff Engineer, joined 18 months ago" + concrete porque); (9) **Day-in-the-life content** (linked engineering blog ou video curto); (10) **Engineering blog cross-link** prominent; (11) **GitHub presence** prominent (cf. `github-presence` Sec 4 employer brand 2026). **WEEK 4 — Roles + growth pathways**: (12) **Honest job titles** (NAO "Ninja Engineer" — cringe 2026); (13) **Growth pathway clear** ("Senior → Staff in 18-30 months typical, with X criteria"); (14) **Success at 90 days** concrete ("shipped 1 feature, onboarded 2 customers, contributed RFC"); (15) **Success at 1 year** concrete ("led major project, hired 1 person, owns subsystem X"); (16) **Compensation philosophy** documentado public (transparent banding system). **WEEK 5 — Process + UX**: (17) **Mobile-first** application (60-70% candidates apply mobile); (18) **Form 3 fields max** (name, email, why interested) — NAO 30 fields; (19) **Process transparent** mapped 4-6 stages com timeline (e.g., "21-28 days typical"); (20) **What to expect per stage**; (21) **Honest "we'll respond in 7 days"** (NAO "we'll be in touch if interested" — soul-crushing); (22) **Take-home alternative** offered (paid se long); (23) **References + offer timeline**. **WEEK 6 — Roll-out + iterate**: (24) Announce internally first (current engineers feedback last round); (25) Soft launch — A/B test new vs old page com candidates 30 dias; (26) Monitor metrics (apply rate, drop-off form, time-to-apply, source quality); (27) Post-30 days iterate baseado em feedback. **CULTURE expression CRITICAL 2026** (Sec 8.5 exemplo): NAO platitudes ("we value teamwork") MAS specifics ("we async-first 4 hours/week sync max + Crucial Conversations framework + quarterly offsites covered"). Audiencia sees thru platitudes immediately. **Anti-padroes IMPORTANTES**: (a) "family" remove (red flag senior — implies overwork expectations); (b) "rockstars/ninjas/wizards" remove (cringe); (c) "work hard play hard" remove (red flag); (d) "competitive salary" sem range (cynism candidate); (e) "unlimited PTO" without context (encoraja menos PTO inadvertently); (f) generic diversity statement (lip service detected); (g) overly polished page sem real culture sinais (slick = "hiding something"). **ROI esperado 90 dias**: applicant quality lift +30-50% (filter natural — engineers serious sobre opportunity research deeply); time-to-hire reduction 20-30%; offer acceptance rate lift 15-25%; employer brand recognition lift in surveys. **Investment**: rewrite content + design + development = $10-30k OR 4-8 sem in-house team time. **Caveat**: changes alone don't fix culture mismatches — if culture realmente "family" no negative sense (overwork), copy adjustment ineffective long-term. **Cross-ref `composicao-visual` Sec 9** mascot/avatar consistency + `github-presence` Sec 4** organization page employer brand. **Quick win imediato (1 sem)**: salary ranges added prominently + remove "family/rockstars" wording + 1 real engineer quote replace 1 stock photo = signal massive sem rewrite total.

**Comportamento-chave:** diagnostico red flags 2026 multiple, plano 6 semanas detalhado em 6 fases, mobile-first specific, anti-padroes especificos, ROI projection + investment, quick win 1 semana.
