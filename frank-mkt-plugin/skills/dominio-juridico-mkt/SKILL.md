## 3. LGPD profundo (extraido frank_juridico_garden Skill 09)

### 3.1 15 documentos obrigatorios/recomendados LGPD

> **Source**: `frank_juridico_garden/core/skills/09-compliance-lgpd.md` Sec 2.1

```
DOCUMENTOS LGPD 2026 (Brazilian)

1. POLITICA DE PRIVACIDADE (externa)
   - Obrigatorio: SIM
   - Base legal: Arts 6, 9, 18 LGPD
   - Funcao: informa titular finalidade + base legal + direitos

2. POLITICA INTERNA PROTECAO DADOS
   - Recomendado (pratica de mercado)
   - Base legal: Art 50
   - Funcao: regras internas colaboradores

3. ROPA — REGISTRO OPERACOES TRATAMENTO (Record of Processing Activities)
   - Obrigatorio: SIM
   - Base legal: Art 37
   - Funcao: mapeamento todos tratamentos dados pessoais

4. RIPD — RELATORIO IMPACTO PROTECAO DADOS
   - Obrigatorio: SIM (quando exigido ANPD ou alto risco)
   - Base legal: Art 38
   - Funcao: analise riscos + medidas mitigatorias

5. TERMO CONSENTIMENTO
   - Obrigatorio: SIM (quando base legal = consentimento)
   - Base legal: Arts 7, I; 8; 11, I
   - Funcao: manifestacao livre + informada + inequivoca titular

6. AVISO COOKIES
   - Recomendado
   - Base legal: Arts 7, 8
   - Funcao: informa cookies + solicita consentimento

7. CLAUSULA PROTECAO DADOS CONTRATOS
   - Obrigatorio: SIM
   - Base legal: Arts 39, 42, 46
   - Funcao: tratamento dados em contratos

8. DPA — DATA PROCESSING AGREEMENT
   - Recomendado
   - Base legal: Arts 39, 42
   - Funcao: acordo especifico operadores

9. POLITICA RETENCAO/DESCARTE DADOS
   - Recomendado
   - Base legal: Art 16
   - Funcao: prazos retencao + procedimento eliminacao

10. PLANO RESPOSTA INCIDENTES SEGURANCA
    - Obrigatorio: SIM
    - Base legal: Art 48
    - Funcao: identificar + conter + comunicar + remediar vazamentos

11. POLITICA SEGURANCA INFORMACAO
    - Recomendado
    - Base legal: Art 46
    - Funcao: medidas tecnicas + administrativas

12. REGISTRO INCIDENTES SEGURANCA
    - Obrigatorio: SIM
    - Base legal: Art 48
    - Funcao: log incidentes + acoes tomadas

13. TERMO RESPONSABILIDADE/CONFIDENCIALIDADE colaboradores
    - Recomendado
    - Base legal: Art 46
    - Funcao: compromisso colaboradores

14. NOMEACAO DPO (Encarregado)
    - Obrigatorio: SIM
    - Base legal: Art 41
    - Funcao: designacao formal publicada site

15. FORMULARIO ATENDIMENTO DIREITOS TITULAR
    - Obrigatorio: SIM
    - Base legal: Art 18
    - Funcao: canal exercicio direitos (acesso, correcao, eliminacao, portabilidade)
```

### 3.2 10 bases legais Art 7 LGPD

> **Source**: `frank_juridico_garden/skills/09-compliance-lgpd.md` Sec 2.2

```
BASES LEGAIS PARA TRATAMENTO DADOS PESSOAIS

I. CONSENTIMENTO (Art 7, I)
   Quando: NAO houver outra base mais adequada
   Exemplo: newsletter, marketing direto, compartilhamento parceiros
   Caracteristicas: livre + informado + inequivoco; revogavel

II. OBRIGACAO LEGAL/REGULATORIA (Art 7, II)
   Quando: lei exige
   Exemplo: Marco Civil registros acesso, escrituracao, eSocial
   Caracteristicas: NAO depende consentimento

III. ADMINISTRACAO PUBLICA (Art 7, III)
   Quando: orgaos publicos politicas publicas
   Exemplo: SUS, Bolsa Familia, Censo
   Caracteristicas: setor publico exclusivo

IV. PESQUISA (Art 7, IV)
   Quando: orgao pesquisa
   Exemplo: IBGE, universidades, pesquisa clinica
   Caracteristicas: anonimizacao quando possivel

V. EXECUCAO CONTRATO (Art 7, V)
   Quando: tratamento necessario cumprir contrato titular
   Exemplo: processar pagamento, entregar produto
   Caracteristicas: a pedido titular ou pre-contratual

VI. EXERCICIO REGULAR DIREITOS (Art 7, VI)
   Quando: processo judicial/administrativo/arbitral
   Exemplo: guarda documentos defesa, producao provas
   Caracteristicas: inclui pre-processual

VII. PROTECAO VIDA/INCOLUMIDADE FISICA (Art 7, VII)
   Quando: emergencias
   Caracteristicas: aplicacao restrita

VIII. TUTELA SAUDE (Art 7, VIII)
   Quando: profissionais saude
   Caracteristicas: exclusivamente saude/sanitaria

IX. LEGITIMO INTERESSE (Art 7, IX)
   Quando: interesse legitimo controlador
   Exemplo: prevencao fraude, seguranca rede, marketing direto clientes
   Caracteristicas: exige LIA (Legitimate Interest Assessment); MAIS contestada

X. PROTECAO CREDITO (Art 7, X)
   Quando: analise risco credito
   Exemplo: SPC/Serasa, score credito
   Caracteristicas: especifica direito brasileiro
```

|--------------|---------------|

| **AI-generated juridico content** | Hallucinations frequentes (precedentes inventados) |
| **Disclaimer ausente** | Liability + audience confusion |
| **Confidential client data** | LGPD violation + ethics |
| **Tabelas honorarios publicas** (advogado) | OAB Provisao 205 |
| **Comparacao com colegas** (advogado) | OAB Provisao 205 |
| **Captacao agressiva** (advogado) | OAB violations |
| **Lei 15.325 disclosure absent** influencer post | Lei 15.325 + CDC violations |
