# Skill: brand-book-methodology

## Decaimento Temporal

- volatility: medium
- last_review: 2026-05-09
- next_review: 2026-11-09

MAS camada digital evolui rapido: Frontify + Zeroheight lancaram features AI-queryable em 2025-2026, design tokens W3C Community Group fechou primeira spec estavel em 2024, design systems migraram de PDF para living documents (Airbnb DLS 2014 -> Spotify Encore 2022 -> Material 3 Expressive 2025). ## Visao Geral

Esta skill organiza a **metodologia de construcao do Brand Book** (manual de identidade visual + verbal) como artefato vivo e governado, nao como PDF estatico de 80 paginas que ninguem le. Brand book e o documento fundacional que separa marca **consistente** de marca **bagunca-no-marketplace**.

**Diferenca conceitual critica** (fonte: Frontify + Bynder + IxDF 2026):
- **Brand book** = documento estrategico-narrativo-visual (proposito + missao + valores + storytelling + personalidade + tom de voz + identidade visual). Responde "QUEM somos".
- **Brand guidelines / Style guide** = manual tatico operacional (especificacoes tecnicas: clear-space do logo, hex das cores, kerning das tipografias). Responde "COMO mostramos".
- **Design system** = camada de implementacao (componentes UI + design tokens + bibliotecas Figma). Responde "COMO construimos digital".

Brand book moderno (2026) **engloba os 3** em camadas hierarquicas: storytelling no topo + specs no meio + design system (quando aplicavel) na base, **conectados** via plataforma digital (Frontify/Zeroheight/Bynder).

**5 secoes core canonicas** (consenso Akrivi + Kijo + Kedraco + Frontify + Ebaq + Venngage 2026):
1. **Marca** (logo + variations + clear-space + don'ts + simbolo + wordmark)
2. **Cores** (paleta primaria + secundaria + neutras + acessibilidade + aplicacoes)
3. **Tipografia** (display + texto + hierarquia + escala + pareamento)
4. **Aplicacoes** (digital + print + co-branding + embalagem + signage + merchandise)
5. **Don'ts** (uso incorreto + dilution + violations + casos reais)

**3 secoes extras 2026** (modernidade pos-2020):
6. **Tom de voz + verbal identity** (vocabulary + boilerplate + microcopy + storytelling)
7. **Iconografia + ilustracao + fotografia** (estilo visual estendido)
8. **Motion + sound + acessibilidade WCAG** (digital-first era)

Esta skill e para **brand designers** entregando manual completo, **CMO/CCO** governando consistencia em escala, **agencias** padronizando entregavel premium, **design-ops** estruturando design systems conectados, e **design-leads** treinando time interno + terceiros (agencias parceiras + freelancers + influenciadores). Segunda skill do Bloco Identidade Visual Corporativa / Brand Design — sister direta de logo-design-process + paleta-cores-corporativa + tipografia-corporativa + iconografia-corporativa.

## Fundacao 1 — Estrutura Canônica Brand Book (5 Seções Core + 3 Extras)

### Hierarquia padrao 2026

```
BRAND BOOK
|
+-- 0. Capa + Metadata (versao + data + responsavel + contato)
+-- 1. Brand Story (proposito + missao + visao + valores + personalidade)
+-- 2. MARCA (logo + variations + clear-space + don'ts)
+-- 3. CORES (paleta + acessibilidade + aplicacao)
+-- 4. TIPOGRAFIA (display + body + hierarquia + escala)
+-- 5. APLICACOES (digital + print + co-branding + embalagem + signage)
+-- 6. DON'TS (galeria de uso incorreto)
+-- 7. (extra) Tom de Voz + Verbal Identity
+-- 8. (extra) Iconografia + Ilustracao + Fotografia
+-- 9. (extra) Motion + Sound + Acessibilidade
+-- 10. Governanca (versioning + contato + licenciamento + downloads)
```

### Brand Book vs Style Guide vs Design System (matriz comparativa)

| Aspecto | Brand Book | Style Guide | Design System |
|---------|-----------|-------------|---------------|
| **Foco** | Estrategico + narrativo | Tatico + tecnico | Implementacao + componentes |
| **Pergunta** | Quem somos? | Como mostramos? | Como construimos digital? |
| **Audiencia** | C-level + colaboradores + parceiros | Designers + agencias + impressores | Devs + designers digital |
| **Formato** | PDF + portal digital + livro impresso | PDF + portal | Codigo + Figma + portal |
| **Tamanho tipico** | 80-150 paginas | 30-60 paginas | Library + docs vivos |
| **Atualizacao** | 2-5 anos (rebranding) | Anual ou semestral | Continua (sprints) |
| **Exemplos** | Mailchimp Patterns + Slack Brand | NASA 1976 + USPS DMM | Material 3 + Carbon + HIG |

### Ordem narrativa recomendada

**Errado** (comum): comecar por logo, cor, tipografia (specs tecnicas) e depois voltar pra story. Resultado: leitor nao entende **por que** as escolhas foram feitas, vai aplicar mecanico.

**Certo**: comecar por **Brand Story** (proposito + valores) e depois **conectar cada elemento visual ao porque**. Cor primaria nao e "azul porque e bonito" — e "azul porque transmite confianca, alinhado ao valor 'transparencia' da marca".

Conexao narrativa = **mortar** que liga os tijolos. Sem ela, brand book vira catalogo de specs.

## Fundacao 2 — Seção Marca (Logo + Variations + Don'ts + Espaçamento)

A secao Marca e tipicamente a **maior** (20-25% das paginas) e a mais consultada operacionalmente. Estrutura canonica:

### 2.1 Construcao + grid + proporcoes

- **Grid de construcao**: mostra logo em malha (com circulos/quadrados de referencia) — comprova precisao matematica + permite reconstrucao
- **Proporcoes geometricas**: razao aurea (1:1.618), proporcao 1:2, modulo de unidade x
- **Anatomia do logo**: nomeacao das partes (simbolo + wordmark + tagline + descritor)

### 2.2 Variations (variacoes oficiais)

Tipicamente 5-8 variacoes:
1. **Logo completo positivo** (simbolo + wordmark, fundo claro)
2. **Logo completo negativo** (simbolo + wordmark, fundo escuro)
3. **Logo monocromatico** (1 cor, qualquer fundo)
4. **Simbolo isolado** (so o icone, para favicon + app icon)
5. **Wordmark isolado** (so o tipo, para situacoes onde simbolo nao cabe)
6. **Logo horizontal** (formato 16:9 friendly)
7. **Logo vertical/empilhado** (formato 1:1 friendly)
8. **Logo reduzido** (versao simplificada para mobile/favicon)

Cada variacao deve listar:
- Quando usar (contexto especifico)
- Tamanho minimo (px digital + mm impresso)
- Formatos disponiveis (SVG + PNG + EPS + PDF)
- Hash do arquivo oficial (para versionamento)

### 2.3 Clear-space (area de protecao)

Regra **inegociavel**: zona livre ao redor do logo sem nenhum outro elemento. Tipicamente definida como **multiplo de unidade x** (altura da letra inicial ou de modulo do simbolo).

Exemplos benchmark:
- **NASA 1976**: clear-space = 1x altura do "N"
- **Apple HIG**: clear-space = 1/2 altura do logo Apple
- **Google**: clear-space = altura do "G" minusculo
- **Itau (Brasil)**: clear-space = 1x modulo do simbolo

Visualizar com retangulo tracejado ao redor do logo + cota com setas.

### 2.4 Tamanho minimo

Definir **menor tamanho legivel**:
- **Digital**: 24px altura (favicon padrao 16x16 muitas vezes pede versao reduzida)
- **Impresso**: 10mm altura tipico (cartao de visita) — 5mm para versao reduzida
- **Outdoor**: nao tem maximo, mas validar viewbox SVG

### 2.5 Don'ts especificos do logo

Esta micro-secao dentro de Marca + secao 6 dedicada a Don'ts global. Padrao 6-12 itens:
1. Nao distorcer (estirar/comprimir)
2. Nao rotacionar fora dos angulos oficiais
3. Nao trocar cores
4. Nao adicionar efeitos (sombra, gradient, glow)
5. Nao colocar sobre fundos de baixo contraste
6. Nao recortar/cropar
7. Nao reposicionar elementos (mover simbolo separado do wordmark)
8. Nao redesenhar / recriar (sempre usar arquivo oficial)
9. Nao usar versao defasada (sempre versao current)
10. Nao colocar dentro de caixas/bordas decorativas

Cada don't visualizado com **X vermelho** sobre o exemplo errado + texto explicativo.

### 2.6 Co-branding + endorsement

Quando logo aparece com outras marcas:
- **Lockup horizontal**: separador "+" ou "x" + 1x clear-space entre marcas
- **Endorsement vertical**: marca principal acima + "powered by" + marca parceira
- **Hierarchy**: nunca duas marcas com mesmo peso visual (uma sempre lider)

### 2.7 Aplicacao em fundos complexos

- **Fundos coloridos**: usar versao monocromatica
- **Fundos fotograficos**: validar contraste WCAG AA (4.5:1 minimo) — em ultimo caso, adicionar "scrim" (overlay escuro 40% opacity)
- **Fundos pattern**: nunca colocar logo direto sobre pattern — sempre area limpa primeiro

## Fundacao 3 — Seção Aplicações (Digital + Print + Co-branding + Embalagem)

Secao mais **operacional** (25% paginas). E onde a marca **encontra o mundo real**. Estrutura:

### 3.1 Digital (mandatorio 2026)

- **Website**: header + footer + landing page templates
- **Social media**: avatares (Instagram/LinkedIn/Twitter/TikTok/YouTube) + capas + posts (1:1 + 4:5 + 9:16) + stories + reels
- **Email signature**: padrao corporativo + assinaturas tematicas
- **Apresentacoes (PPT/Keynote)**: templates 16:9 + slide master + galeria de slides
- **App icons**: iOS (1024x1024) + Android (Adaptive Icons 432x432) + favicon (16/32/48)
- **Banner ads**: tamanhos IAB padrao (300x250 + 728x90 + 160x600 + 320x50)
- **Newsletter**: header + footer + cores + tipografia adaptada email-safe

### 3.2 Print

- **Cartao de visita**: frente + verso + dimensoes (90x50mm Brasil padrao, 85x55mm internacional)
- **Papelaria**: papel timbrado A4 + envelope (DL/C5/C4) + envelope janela
- **Folder/folheto**: tipicos 3-fold + 4-fold + book-fold
- **Outdoor**: 9x3m padrao Brasil + busdoor + abrigo
- **Anuncio impresso**: formatos jornal/revista
- **Brochura/catalogo**: capa + miolo + gramaturas papel sugeridas

### 3.3 Embalagem (quando aplicavel)

- **Caixa primaria**: dieline + zonas seguras + posicionamento logo
- **Rotulo**: requisitos legais (Brasil: Anvisa para alimentos/cosmeticos + Inmetro para outros)
- **Sacola/sacaria**: variacoes papel + plastico + reciclavel
- **Etiqueta de produto**: codigo de barras + posicionamento

### 3.4 Signage + ambiental

- **Fachada**: letras caixa + ACM + acrilico + iluminacao
- **Sinalizacao interna**: placas de sala + wayfinding + diretorios
- **Sinalizacao externa**: totem + monoposte + fachada
- **Ambient branding**: paredes + adesivos + decoracao

### 3.5 Merchandise + brindes

- **Vestuario**: camiseta + boné + jaqueta (pantone for textile)
- **Estacionario**: caderno + caneta + agenda
- **Tech**: capa de notebook + adesivo + mouse pad
- **Eventos**: banners + backdrop + crachas + lanyards

### 3.6 Veicular

- **Carro corporativo**: laterais + traseira + capo
- **Frota de delivery**: mototaxis + vans + caminhoes
- **Sinalizacao veicular**: placa magnetica + adesivo definitivo

### 3.7 Co-branding (matriz de aplicacao)

| Cenario | Hierarquia | Exemplo |
|---------|-----------|---------|
| **Sponsor** | Marca lider 100% + sponsor 60-70% | Evento Itau Cultural + bandas |
| **Joint Venture** | 50-50 lockup | Stone + Linx (fusao 2021) |
| **White-label** | Marca cliente 100% + nossa "powered by" 30-40% | Banco como Servico |
| **Endorsement** | Marca-mae 80% + sub-marca 100% | Magalu + Cancao Nova |
| **Partnership** | 50-50 horizontal "x" separador | Nubank x Mastercard |

### 3.8 Templates editaveis

Brand book moderno **disponibiliza** templates Figma/PPT/InDesign com:
- Master slides + componentes pre-configurados
- Bibliotecas com logo + cores + tipografia
- Permissoes (so consulta vs edicao + duplicacao)
- Versionamento integrado

## Fundacao 4 — Ferramentas Digital (Frontify + Zeroheight + Figma + InDesign)

Brand book 2026 **nao e mais PDF estatico** — e plataforma viva. Estado da arte:

### 4.1 Frontify (lider em brand portals)

- **Foco**: brand management + DAM + portal corporativo + design system unificado
- **Ideal para**: marketing teams + brand designers + enterprise (50+ pessoas)
- **Forca**: customizacao profunda (white-label do portal) + DAM nativo + workflows de aprovacao
- **Modulos**: Guidelines + DAM + Templates + Print Templates + Workspace
- **Pricing 2026**: a partir de USD 89/usuario/mes (Enterprise sob consulta)
- **Cases**: Vodafone + Lufthansa + Allianz + KitKat + Hyundai
- **Site**: [frontify.com](https://www.frontify.com/)

### 4.2 Zeroheight (lider em design systems docs)

- **Foco**: design system documentation + dev handoff + token sync
- **Ideal para**: product/engineering teams + tech companies + startups
- **Forca**: integracao Figma nativa + Storybook + GitHub + Tokens W3C
- **Modulos**: Design System Pages + Component Library + Tokens + Versioning
- **Pricing 2026**: free tier + a partir de USD 75/editor/mes
- **Cases**: AbbVie + Booking.com + Mass General Brigham + Ferrari
- **Site**: [zeroheight.com](https://zeroheight.com/)

### 4.3 Figma (design source of truth)

- **Modulo Brand**: Figma Files + Components + Variables + Styles
- **Variables (2024+)**: design tokens nativos + multi-mode (light/dark/brand)
- **Library publishing**: bibliotecas centralizadas + versioning + diff visual
- **Plugins essenciais**: Tokens Studio + Stark (acessibilidade) + Brandfetch + Iconify
- **Limitacao**: e source of truth de **specs** mas nao substitui storytelling estrategico

### 4.4 Adobe InDesign + Adobe Express (legado + creator)

- **InDesign**: produzir versao **PDF + impresso** do brand book (livro fisico tipico)
- **Adobe Express (2024+)**: democratiza criacao de assets on-brand para colaboradores nao-design
- **Adobe Assets**: DAM nativo Adobe (alternativa a Frontify para empresas Adobe-stack)

### 4.5 Bynder (alternativa enterprise)

- **Foco**: DAM enterprise + brand guidelines + creative workflow
- **Cases**: Spotify + Puma + Five Guys
- **Forca**: DAM mais maduro + integracao com Salesforce/HubSpot/Adobe

## Fundacao 5 — Governança + Versioning + Uso Por Terceiros

Sem governanca, brand book **deriva em 6 meses** (Marq Brand Governance Report 2026). Framework canonico:

### 5.1 Modelo de governanca distribuida

```
+--- Brand Director / Head of Design (guardiao primario)
|     |
|     +--- Brand Council (mensal, decisoes estrategicas)
|           - CMO + CCO + Product Lead + Eng Lead
|
+--- Brand Champions (rede capilar)
|     - 1 champion por equipe major (vendas + marketing + RH + produto)
|     - Treinamento trimestral + Slack channel #brand-questions
|
+--- Brand Ops (execucao diaria)
      - Aprovacao assets + manutencao DAM + versionamento
```

### 5.2 Versioning semantico do brand book

Aplicar **SemVer 2.0** ao brand book:
- **MAJOR** (X.0.0): rebranding completo (logo novo + paleta nova)
- **MINOR** (1.X.0): adicao de novas secoes ou variacoes (ex: novo logo de produto + paleta secundaria)
- **PATCH** (1.1.X): correcoes + ajustes finos (ex: fix typo + clarificacao don't)

Cada release acompanha:
- **CHANGELOG.md** com diff visual
- **Migration guide** para times atualizarem assets
- **Deprecation period** (60-90 dias para fase-out de versao antiga)
- **Hash de cada arquivo** (SHA-256) para validar integridade

### 5.3 Asset deprecation lifecycle

```
ATIVO (current) -> DEPRECATED (uso permitido + warning) -> ARCHIVED (uso proibido + historico)
   |                       |                                       |
   3 meses                 60-90 dias                              indefinido
   notice                  grace period                            audit trail
```

### 5.4 Uso por terceiros (matriz de licenciamento)

Definir explicitamente **quem pode usar + para que + como**:

| Terceiro | Pode usar logo? | Pode usar cores? | Aprovacao? | Materiais |
|----------|----------------|-------------------|------------|-----------|
| **Funcionarios** | Sim (assets internos) | Sim | N/A | Brand portal full access |
| **Agencias parceiras** | Sim (com NDA + aprovacao) | Sim | Por projeto | Subset + tracker |
| **Influenciadores** | Limitado (lockup oficial co-branded) | Limitado | Por campanha | Press kit dedicado |
| **Imprensa** | Sim (logo oficial only) | Nao para criar assets | Cobertura editorial OK | Press kit publico |
| **Investidores/M&A** | Sim (em decks oficiais) | Sim | Caso a caso | Subset NDA |
| **Concorrentes** | Nao | Nao | N/A | Cease & desist |
| **Comunidade open-source** | Caso a caso | Caso a caso | Caso a caso | Trademark policy |

### 5.5 Press kit publico (boa pratica)

Criar `/press` ou `/brand` no site oficial com:
- Logos (SVG + PNG transparente + JPG fundo branco)
- Fotos oficiais executivos
- Boilerplate company description (50/100/200 palavras)
- Contato imprensa
- Termos de uso (link)

Evitar download manual + email request — fricao desnecessaria.

### 5.6 Compliance check automatizado

Tooling 2026:
- **Brandfetch API**: detecta logo de empresa + valida fidelidade
- **Frontify Compliance Scanner**: monitora assets em redes sociais
- **Custom Lighthouse rules**: valida brand colors em produto digital
- **Reverse image search alerts**: TinEye + Google Alerts para uso indevido

### 5.7 Trademark + IP protection

- **Registro INPI Brasil** (R$ 355 + R$ 745 por classe Nice 11) — proteccao 10 anos renovavel
- **Madri Protocol** (internacional) — registro multi-paises via WIPO
- **Trademark policy publica** (em /trademark): define use justo + use proibido
- **Monitoring service**: Markify, Alegis, Trademarkia (USD 50-300/mes)

### 5.8 Onboarding + treinamento

- **Quarter 1 ano novo colaborador**: workshop 90min brand book
- **Anual refresher**: quiz + atualizacoes
- **Microlearnings mensais**: 1 dica via Slack + e-mail interno
- **Office hours brand**: 1h/semana espaco aberto para tirar duvidas

### 6.2 Casos governo Brasil

- **Governo Federal "Brasil Uniao e Reconstrucao" 2023**: manual publico + verde+amarelo+azul renovado
- **GovBR (digital)**: design system publico (gov.br/ds) + open source
- **Fundacao Banco do Brasil**: manuais publicos das marcas FBB

### 6.3 LGPD + brand content

Brand book 2026 deve incluir secao **LGPD compliance** quando aplica conteudo:
- **Tom de voz** em comunicacoes envolvendo dados (consentimento + opt-in)
- **Boilerplate disclaimers** padrao (rodape de e-mails marketing)
- **Microcopy** em formularios (consentimento explicito + finalidade)
- **Data brand transparency**: pagina dedicada `/privacidade` com voz da marca
- **Direitos do titular**: como atender com tom alinhado (nao corporativo robotico)
- **DPO contact**: incluir no brand book como ponto oficial

### 6.4 Diversidade + acessibilidade Brasil

- **Representatividade fotografica**: brand book deve definir guidelines de inclusao racial/etnica/genero/regional brasileira
- **Acessibilidade visual**: WCAG 2.2 AA minimo + Lei Brasileira de Inclusao 13.146/2015
- **Linguagem neutra/inclusiva**: definir posicionamento (uso ou nao de "todes" + "x") no tom de voz
- **Tradutor LIBRAS**: brand book deve definir aplicacao de janela LIBRAS em videos

### 6.5 Especificidades culturais Brasil

- **Calendario brasileiro**: Natal + Carnaval + Junho festas + Black Friday + dia maes/pais
- **Regionalidade**: Nordeste vs Sul vs Centro-Oeste — consideracoes em campanhas
- **Codigo de Defesa do Consumidor**: linguagem em comunicacao publicitaria (Lei 8.078/1990)
- **CONAR**: autoregulamentacao publicitaria (codigo etico brasileiro)
- **Anvisa + Inmetro**: rotulagem mandatoria por categoria

## Fundacao 7 — Mensuração + Tools 2026

Brand book gera valor mensuravel — mensure ou perde verba.

### 7.1 KPIs de adocao

- **Adoption rate**: % times usando brand portal mensalmente (target: 80%+)
- **Asset reuse rate**: % assets baixados vs criados-do-zero (target: 70%+)
- **Time-to-first-asset**: novo colaborador acessa primeiro asset (target: D+1)
- **Approval cycle time**: media tempo aprovacao asset (target: < 48h)
- **Compliance score**: % assets em conformidade auditoria (target: 95%+)

### 7.2 KPIs de impacto na marca

- **Brand consistency score**: avaliacao trimestral interna (NPS interno) — escala 0-10
- **Recognition recall**: pesquisa publico-alvo lembranca espontanea + assistida
- **Brand health tracking**: surveys + sentiment + share of voice
- **Asset performance**: which formats convert best (A/B testing on-brand)

### 7.3 Ferramentas mensuracao 2026

- **Frontify Analytics** + **Bynder Insights** (DAM nativo)
- **Google Analytics + GA4** (asset downloads)
- **Brandwatch / Talkwalker** (brand mentions + sentiment)
- **Brand24** (alternativa SMB)
- **Lighthouse + axe DevTools** (acessibilidade auditoria)
- **YouGov BrandIndex / Kantar** (brand health Brasil)
- **CONAR** (compliance publicitario auditavel)

### 7.4 Audit trail + compliance log

- Cada modificacao de asset = log com (quem + quando + o que + por que)
- Versioning Git-like (commit history visivel)
- Approval workflow auditavel (aprovador + data + comentario)
- Retencao 5 anos minimo (sugestao alinhada CDC + LGPD)

- **Vocabulary lists**: palavras a usar + palavras a evitar
- **Tone matrix**: serio vs leve / formal vs informal / em qual canal
- **Microcopy guidelines**: botao CTA + mensagens erro + onboarding
- **Boilerplate**: descricoes empresa em 50/100/200/500 palavras

## Anti-Patterns 18

1. **PDF de 200 paginas que ninguem le** — brand book deve ser navegavel + pesquisavel + acionavel
2. **"Atualizo depois"** — sem governanca + versioning, drift garantido em 6 meses
3. **Nao incluir Brand Story** — manual so com specs vira catalogo, sem alma
4. **So uma variacao do logo** — vida real exige multiplas (horizontal + vertical + simbolo + monocromatico minimo)
5. **Nao definir clear-space** — primeira coisa a ser violada na pratica
6. **Cor primaria sem hex/RGB/CMYK/Pantone** — cada midia exige spec diferente
7. **Tipografia premium sem licenca corporativa** — risco juridico (Adobe + Monotype + Linotype rastreiam)
8. **Pouco ou nenhum Don't** — galeria de erros e o que mais ensina + previne
9. **Nao validar acessibilidade WCAG** — ilegal Brasil (LBI 13.146/2015) + perde 17% populacao com deficiencia
10. **Brand book em ferramenta proprietaria sem export** — vendor lock-in arrisca patrimonio
11. **Nao versionar arquivos** — ninguem sabe qual e a versao oficial atual
12. **Distribuir via email + Drive sem governanca** — caos garantido + violacao auditoria
13. **Nao treinar terceiros (agencias + freelancers)** — terceiros violam por ignorancia
14. **Tom de voz vago ("amigavel + serio")** — sem exemplos concretos = inutilizavel
15. **Aplicacoes faltando contextos reais** — so mostrar logo no fundo branco e abstrato
16. **Nao definir co-branding rules** — primeira parceria vira disputa de hierarquia
17. **Press kit ausente / ruim** — imprensa pega logo errado em Google Images
18. **Nao revisar 2-3 anos** — brand books envelhecem, refresh mantem relevancia

## 18 Regras de Ouro

1. **Brand Story SEMPRE primeiro** — conexao narrativa antes de specs
2. **Cada elemento visual conectado a um valor** — explicitar o "por que"
3. **5 secoes core + 3 extras = 80-110 paginas** — abaixo e raso, acima e bloat
4. **Logo: minimo 5 variacoes oficiais** — horizontal + vertical + simbolo + mono pos + mono neg
5. **Clear-space sempre como multiplo de modulo x** — proporcao escala com tamanho
6. **Cores: hex + RGB + CMYK + Pantone + RAL (signage)** — todas as midias cobertas
7. **Tipografia: licenca corporativa documentada + alternativa websafe** — evitar gap juridico
8. **Don'ts visualizados com X vermelho + texto explicativo** — galeria pratica
9. **Aplicacoes minimo: digital + print + social + co-branding** — cobertura basica garantida
10. **Tom de voz com vocabulary do/dont + exemplos concretos** — operacionalizavel
11. **Acessibilidade WCAG AA minimo + LBI 13.146/2015 Brasil** — inegociavel
12. **Versioning SemVer 2.0 + CHANGELOG.md** — rastreabilidade total
13. **Plataforma digital + PDF estatico** — dois formatos, uso complementar
14. **Press kit publico em /brand ou /press** — sem fricao para imprensa + parceiros
15. **Trademark policy publica + INPI Brasil registrado** — protecao juridica
16. **Onboarding obrigatorio novos colaboradores** — workshop 90min + quiz
17. **Audit trail + retencao 5 anos minimo** — compliance LGPD + CDC
18. **Revisao a cada 2-3 anos** — refresh para evitar obsolescencia

## Exemplos Comportamentais

### Persona 1 — Carolina, CMO de fintech Series B (Sao Paulo)

**Contexto**: 120 funcionarios + 3 agencias parceiras + crescimento 15% MoM. Brand foi definido em 2022 mas "esta apagando". Time de marketing reclama que cada designer interpreta diferente. Investidores cobram consistencia para Series C.

**Skill ativada**: Carolina aplica fundacoes 1 + 4 + 5 + 7. Audita brand book atual (PDF 60p Drive) — confirma que esta defasado + ninguem acessa. Decide migrar para Frontify (USD 2.5k/mes inicial). Aloca brand-ops 1 FTE por 6 meses para construir portal + atualizar assets. Define governanca distribuida: ela como guardiao + 4 brand champions (1 por equipe). Implementa SemVer + CHANGELOG. Aplica fundacao 7: define KPI adoption rate (target 80% em 6m), compliance score (95% target), brand consistency score interno trimestral. Apos 6 meses: portal vivo + adoption 85% + Series C captou USD 80M com brand consistency citado por investidores.

**Output**: investimento USD 60k em 12 meses (tooling + FTE + relauch). Ganho: brand consistency mensuravel + agencias parceiras alinhadas + asset reuse rate 75% (era 30% antes). Pitch Series C reforcado.

### Persona 2 — Pedro, freelancer brand designer (Recife)

**Contexto**: agencia solo, 8 anos de mercado, atende ~12 clientes/ano. Cliente PME paga R$ 8-15k por brand identity completa. Quer subir ticket para R$ 25-40k entregando brand book premium.

**Skill ativada**: Pedro aplica fundacoes 1 + 2 + 3 + 6. Define entregavel padrao 80 paginas em 5 secoes core + 2 extras (tom de voz + iconografia). Cria template Figma reutilizavel + master InDesign para gerar PDF impresso. Inclui mock-ups Brasil-relevantes (cartao R$ + sacola Magalu-style + outdoor). Casos brasileiros como referencia (Magalu + Nubank + Stone). Aplica fundacao 5 para entregar com governance light: SemVer + CHANGELOG + 1h workshop com cliente sobre uso. Aplica fundacao 8: tom de voz + microcopy basico. Sobe ticket para R$ 28k mantendo margem 60%.

**Output**: 6 entregas em 10 meses + 2 cases pra portfolio + indicacoes (LTV cliente: R$ 28k -> R$ 60k em 24m com servicos recorrentes brand ops). NPS cliente 9.2.

### Persona 3 — Marina, Design Lead em scale-up (Curitiba, B2B SaaS)

**Contexto**: empresa 250 pessoas + 4 produtos + design system disperso (cada produto tem suas regras). CEO pede "harmonizacao da marca". Marina lidera squad de design (8 designers).

**Skill ativada**: Marina aplica fundacoes 1 + 4 + 7 + 8. Decide ir Zeroheight (foco design system) + Figma Variables. Estrutura brand book em camadas: estrategico (PDF 60p) + style guide (Zeroheight portal) + design system (Figma + Storybook). Aplica fundacao 5: governanca com Brand Council mensal (CMO + CCO + Eng Lead + ela). SemVer rigoroso + deprecation lifecycle 90 dias. Treina 4 brand champions (1 por produto). Aplica fundacao 7: KPIs adoption + compliance + asset reuse + asset performance (A/B em CTAs). Aplica fundacao 8: tom de voz operacional + influencer playbook + content audit trimestral.

**Output**: 9 meses para harmonizar + design system unificado + adoption 90% + cross-product consistency mensuravel. Empresa fecha Series C 4 meses depois.

### Persona 4 — Luiza, agencia (boutique 12 pessoas, Sao Paulo)

**Contexto**: agencia 8 anos especializada em rebranding. Recebe convite Globo para rebranding de produto. Precisa entregar brand book enterprise-grade.

**Skill ativada**: Luiza aplica TODAS as fundacoes 1-8 em entregavel premium. Estrutura 110 paginas + 5 core + 3 extras + governance. Aplica casos brasileiros benchmark (proprio Globo historico + concorrentes). Inclui: tom de voz operacional + influencer kit + co-branding matrix + LGPD content + diversidade Brasil. **Output**: contrato 7 digitos + reconhecimento WPP + 3 indicacoes para projetos enterprise apos 12 meses + livro publicado como case Brasil de design management.

6. **content-mkt** + **tom-de-voz** operacionalizam comunicacao
7. **lgpd-compliance** + **acessibilidade-wcag** garantem compliance transversal

Esta skill e o **hub central** do bloco — sem ela, os outros entregaveis ficam dispersos. E o "container" que organiza decisoes de branding em artefato vivo, governado, mensurado, e usavel por todos os stakeholders (interno + agencias + terceiros + imprensa).
