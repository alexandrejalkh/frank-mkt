## 2. Cenario 2026 — funis adaptive + AI-driven

### 2.1 Stats fundamentais 2026

```
ROI FULL-FUNNEL VS SINGLE-STAGE
   Brands com full-funnel strategy: +45% ROI
   vs competitors single-stage focus
   = invest no funnel COMPLETO, não fragmentos

B2B TOUCHPOINTS
   12+ touchpoints required antes de conversao
   = nurturing strategy mandatory
   = each touchpoint stage-specific content

B2B BUYING CYCLE 2026
   10.1 meses average (vs 11.3 ano antes)
   Economic pressure encurtando
   = velocity matters mais que nunca

FIRST-CONTACT SHIFT
   First sales contact: 61% of journey (vs 69% antes)
   Sellers getting in EARLIER 6-7 weeks
   = top-of-funnel content + thought leadership critical

CUSTOMER JOURNEY ADAPTIVE
   2026 = NOT linear sequence
   Consumers loop between stages
   Mood/research depth/peer influence
   = AI personalization stage-contextual
```

### 2.2 'MQL is dead' shift 2026

```
LEGACY METRICS (2010s)
   MQL (Marketing Qualified Lead)
   Vanity metric = volume sem qualidade
   Marketing optimizes MQL count
   Sales rejects 80% of MQLs as unqualified

2026 SHIFT
   PRIMARY KPIs:
   - SQL (Sales Qualified Lead)
   - Pipeline created (revenue-weighted)
   - CAC payback period
   - Net revenue retention (NRR)

   SECONDARY:
   - Customer engagement score
   - Product activation rate
   - Time-to-value (TTV)
```

### 2.3 PLG (Product-Led Growth) era

```
PLG VS SLG (Sales-Led Growth)

PLG
   Product = primary acquisition + activation + retention + expansion driver
   Free trial / freemium / self-serve
   North Star Metric (NSM) = activation event
   Examples: Slack, Notion, Figma, Linear

SLG
   Sales reps drive each stage
   Demos required + custom pricing
   Sales cycle long (weeks-months)
   Examples: Salesforce, Workday, ServiceNow

HYBRID (most common 2026)
   Self-serve + sales-assisted
   PLG = top of funnel; SLG = mid+bottom
   Examples: HubSpot, Zoom, Atlassian

DECISION MATRIX
   - ACV (Annual Contract Value) low ($5-50k) = PLG
   - ACV high ($50k+) = SLG or hybrid
   - Complex implementation = SLG
   - Simple onboarding = PLG
```

## 3. AIDA + AARRR + AAARRR foundational

### 3.1 AIDA (St. Elmo Lewis 1898)

```
ATTENTION
   Capture audience awareness
   Hooks, headlines, visuals
   Channel: ads, content, social

INTEREST
   Engage curiosity
   Stories, education, value
   Channel: blog, video, lead magnets

DESIRE
   Build emotional + rational want
   Benefits, social proof, demos
   Channel: email, case studies, webinars

ACTION
   Convert to purchase
   Clear CTA, friction removal
   Channel: landing pages, checkout

EXTENSIONS
   AIDCAS (+ Conviction + Satisfaction)
   AIDA-S (+ Satisfaction)
   = post-purchase explicit
```

### 3.2 AARRR Pirate Metrics (Dave McClure 2007)

```
ACQUISITION
   How users find you
   Traffic source, channel, ad spend
   Metric: visitors, signups, downloads

ACTIVATION
   First "aha moment"
   Onboarding completion, key feature use
   Metric: activation rate (% reaching aha)
   = MOST IMPORTANT (predicts retention)

RETENTION
   Users stay engaged
   DAU/WAU/MAU, churn rate
   Metric: cohort retention curves

REFERRAL
   Users invite others
   Viral coefficient, NPS, referral programs
   Metric: viral K-factor (>1 = exponential growth)

REVENUE
   Monetization
   ARPU, LTV, MRR/ARR
   Metric: revenue per acquired user
```

### 3.3 AAARRR (Growth Tribe 2016)

```
+ AWARENESS (added at top)

AWARENESS → ACQUISITION → ACTIVATION → RETENTION → REFERRAL → REVENUE

AWARENESS
   Brand recognition pre-acquisition
   Brand searches, mentions, share-of-voice
   Metric: brand recall, aided/unaided awareness

= MORE COMPLETE for branded businesses
= AARRR alone misses early funnel
```

### 3.4 Decision: AARRR vs AAARRR vs AIDA vs TOFU/MOFU/BOFU

```
AIDA (1898)
   Use: ad creative + sales pitches
   Strength: simple + classic
   Weakness: not measurable easily

AARRR (2007)
   Use: SaaS startups + product analytics
   Strength: action-oriented metrics per stage
   Weakness: skips brand awareness

AAARRR (2016)
   Use: branded businesses + full picture
   Strength: complete funnel
   Weakness: more complex

TOFU/MOFU/BOFU (canonical funnel)
   Use: content marketing + B2B
   Strength: content strategy alignment
   Weakness: ignores post-purchase

RECOMMENDATION 2026
   - SMB SaaS PLG: AARRR (focus on activation)
   - Enterprise B2B: AAARRR + 6-stage journey
   - Content marketing: TOFU/MOFU/BOFU
   - Ad creative: AIDA
   - Combine multiple frameworks per use case
```

## 4. TOFU/MOFU/BOFU canonical

### 4.1 3 estagios

```
TOFU (Top of Funnel)
   Goal: AWARENESS + EDUCATION
   Audience: cold, problem-aware
   Content: blog posts, infographics, social, podcasts,
            "what is X" articles, beginner guides
   Metrics: visitors, impressions, social engagement
   Conversion goal: email signup, follow

MOFU (Middle of Funnel)
   Goal: CONSIDERATION + EVALUATION
   Audience: solution-aware, comparing
   Content: case studies, webinars, ROI calculators,
            comparison guides, free trials, demos
   Metrics: leads, demo requests, content downloads
   Conversion goal: SQL, demo booking

BOFU (Bottom of Funnel)
   Goal: DECISION + PURCHASE
   Audience: vendor-aware, ready to buy
   Content: pricing pages, product demos, free trials,
            customer testimonials, implementation guides
   Metrics: trials, opportunities, closed deals
   Conversion goal: paid customer
```

### 4.2 Content strategy per stage

```
TOFU CONTENT (60-70% effort)
   - Blog posts SEO-optimized
   - Pillar pages (cf. `seo-keywords`)
   - Industry reports
   - Beginner videos
   - Social posts (cf. `multi-platform-narrative`)
   - Newsletter top-of-mind
   - Podcast guesting

MOFU CONTENT (20-25% effort)
   - In-depth guides (3000+ words)
   - Webinars (45-60 min)
   - Case studies (specific outcomes)
   - Comparison tools (vs competitors)
   - ROI calculators
   - Free templates / tools
   - Email nurture sequences (5-12 emails)

BOFU CONTENT (10-15% effort)
   - Product demo videos
   - Customer testimonials (quote + video)
   - Pricing page (transparent)
   - Free trial landing
   - Implementation guides
   - Success metrics per customer
   - Sales enablement content
```

### 4.3 Common mistakes per stage

```
TOFU MISTAKES
   - Selling product (not educating)
   - Generic content competitors also publish
   - SEO with no audience clarity
   - Promo CTAs at top (kills trust)
   - Skipping email capture

MOFU MISTAKES
   - Gating low-value content
   - Pushy nurture emails
   - Generic comparison pages
   - Webinars without action
   - Ignoring lead scoring

BOFU MISTAKES
   - Hiding pricing (B2B 2026 audience expects transparency)
   - Free trial too short (7 days when need 14+)
   - Sales handoff broken (MQL→SQL friction)
   - No urgency (no reason to act NOW)
   - Manual processes for self-serve products
```

## 5. Customer journey B2B vs B2C

### 5.1 Diferencas fundamentais

```
B2B JOURNEY
   Decision-makers: 6-10 buying committee
   Cycle length: 90-180 days SMB / 6-12+ meses enterprise
   2026 average: 10.1 meses
   Touchpoints: 12+ before conversion
   Post-purchase: onboarding + adoption + expansion + renewal
   Risk perception: HIGH (career stake)
   Information depth: deep (whitepapers, RFPs)
   Channels: LinkedIn + email + sales calls + conferences

B2C JOURNEY
   Decision-makers: 1-2 individuals (consumer + maybe partner)
   Cycle length: minutes to days
   Touchpoints: 2-7 typical
   Post-purchase: simpler (use, refer, repurchase)
   Risk perception: LOW (consumer dollars)
   Information depth: shallow (reviews, photos)
   Channels: social + ads + influencers + retail
```

### 5.2 5-stage model (canonical, both B2B + B2C)

```
1. AWARENESS
   Customer learns brand exists
   Triggered by: search, social, ad, referral
   Brand action: TOFU content, advertising

2. CONSIDERATION
   Customer evaluates options
   Comparing 3-5 brands
   Brand action: MOFU content, comparisons

3. DECISION
   Customer commits to one brand
   Final evaluation
   Brand action: BOFU content, demos, trials

4. RETENTION
   Customer uses product
   Onboarding + adoption + ongoing engagement
   Brand action: customer success, support

5. ADVOCACY
   Customer becomes promoter
   Refers others, reviews, UGC
   Brand action: community, referral programs
```

### 5.3 7-stage enterprise B2B

```
1. AWARENESS
   Trigger event (industry change, pain point)

2. INTEREST
   Active research initiation
   Reading content, attending webinars

3. CONSIDERATION
   Vendor longlist (8-15 options)
   Initial filtering

4. INTENT
   Shortlist 3-5 vendors
   Demo requests, RFP issued

5. EVALUATION
   Deep technical/ROI analysis
   Pilot programs, proofs of concept

6. PURCHASE
   Contract negotiation, signature
   Procurement process

7. LOYALTY
   Onboarding → adoption → expansion → renewal
   Multi-year relationship
```

### 5.4 B2B buying committee (6-10 stakeholders)

```
TYPICAL COMMITTEE ENTERPRISE SaaS

DECISION-MAKER (1)
   C-level (CFO, CTO, CEO)
   Final signature
   Cares: ROI, strategic fit, risk

ECONOMIC BUYER (1-2)
   VP/Director with budget
   Cares: cost, business case

CHAMPION (1-2)
   Internal advocate selling internally
   Cares: career success via project

USERS (3-5)
   Engineers/ops/marketers using product
   Cares: usability, productivity

INFLUENCERS (1-3)
   Cross-functional inputs
   Procurement, security, legal, IT
   Cares: compliance, risk, fit

JOURNEY IMPLICATION
   Each persona = different content + concerns
   Multi-thread sales motion
   Cant target only champion (MUST get C-level + economic buyer)
```

## 6. B2B SaaS benchmarks 2026

### 6.1 Conversion rates funil

```
TYPICAL B2B SaaS 2026 (mid-market example)

1000 LEADS (TOFU traffic)
  ↓ (39% MQL conversion)
390 MQLs
  ↓ (38% MQL → SQL — CRITICAL benchmark)
148 SQLs
  ↓ (42% SQL → Opportunity)
62 OPPORTUNITIES
  ↓ (39% close rate from opportunity)
24 CLOSED DEALS

= 2.4% end-to-end conversion (1000 → 24)
= HEALTHY for mid-market B2B SaaS

INDUSTRY-WIDE BENCHMARKS
   MQL → SQL: 38% B2B SaaS (vs 15% cross-industry)
   SQL → Opportunity: 25-50%
   Opportunity → Closed: 15-30% (varies by deal size)
   End-to-end: 1-5% healthy range
```

### 6.2 Win rates 2026

```
ALL DEALS
   B2B average: 21% win rate
   = 79% deals lost (don't close)

QUALIFIED OPPORTUNITIES ONLY
   B2B average: 29% win rate
   = better predictor of pipeline health

BY SEGMENT
   SMB: 25-40% win rate (smaller deals, faster)
   Mid-market: 20-30%
   Enterprise: 10-20% (longer cycles, more competition)

WIN RATE FACTORS
   - Lead qualification rigor
   - Competitive landscape
   - Sales rep skill
   - Product-market fit strength
   - Deal size (larger = lower win rate)
```

### 6.3 Sales cycles 2026

```
SMB (deals <$10k ACV)
   30-60 days typical
   Self-serve sometimes
   1-2 stakeholders

MID-MARKET ($10-100k ACV)
   60-120 days
   Multiple stakeholders
   Demo + trial + custom proposal

ENTERPRISE ($100k+ ACV)
   90-180 days minimum
   Complex buying committee
   Pilot + RFP + procurement + legal
   Sometimes 12+ months

2026 SHIFT
   Average B2B cycle: 10.1 meses (vs 11.3 ano antes)
   Economic pressure forcing faster
   First contact 61% of journey (vs 69% antes)
```

### 6.4 Pipeline coverage

```
PIPELINE COVERAGE = Pipeline value / Quota

COMMON BENCHMARK
   3-5x coverage
   = pipeline 3-5x larger than quota for safety

VARIATION POR WIN RATE
   60% win rate (rare): 1.7x sufficient
   30% win rate: 3.3x typical
   20% win rate: 5x recommended
   15% win rate (enterprise): 6.7x

UNHEALTHY SIGNALS
   <2x: not enough pipeline (will miss quota)
   >7x: too much (low quality, sandbagging, or stalled)
```

### 6.5 CAC + LTV + ratios

```
CAC PAYBACK PERIOD
   Time to recover Customer Acquisition Cost
   Healthy B2B SaaS 2026: 12-18 months
   Excellent: <12 months
   Concerning: 18-24 months
   Bad: >24 months

LTV (Customer Lifetime Value)
   Total revenue per customer over lifetime
   B2B SaaS: 3-5 year typical lifecycle
   = MRR × Avg Lifespan months × Gross Margin

LTV:CAC RATIO
   3:1 minimum healthy
   4-5:1 excellent
   <3:1 = unsustainable
   >5:1 = under-investing in growth (lost opportunity)

GROSS MARGIN B2B SaaS
   70-85% typical
   Higher = better unit economics

NRR (Net Revenue Retention)
   100%+ healthy
   110%+ outstanding
   130%+ best-in-class
   <90% = leaky bucket (churn > expansion)
```

### 6.6 Pipeline velocity formula

```
PIPELINE VELOCITY = (Opportunities × Avg Deal × Win Rate) / Cycle

EXAMPLE
   100 opportunities × $50k avg × 30% win / 90 days
   = $1.67M revenue per day potential

USE
   Identify bottleneck
   Increase opportunities (top of funnel) OR
   Increase deal size (upsell, ACV) OR
   Improve win rate (sales process) OR
   Decrease cycle (velocity programs)
```

### 6.7 12 RevOps KPIs that predict revenue health 2026

```
1. Pipeline velocity
2. Pipeline coverage ratio
3. Win rate
4. Sales cycle length
5. CAC
6. LTV
7. LTV:CAC ratio
8. Net revenue retention (NRR)
9. Forecast accuracy
10. Lead-to-opportunity conversion
11. Quota attainment

## 7. North Star Metric (NSM)

### 7.1 Definicao

```
NORTH STAR METRIC

Single, value-centric measure that:
1. PREDICTS sustainable revenue + retention
2. ALIGNS organization on shared goal
3. CONNECTS to product team efforts
4. NOT vanity metric (visitors, signups)

EXAMPLES NSM (cases)

AIRBNB: nights booked
SPOTIFY: time spent listening
FACEBOOK: monthly active users (MAU) — controversial
SLACK: messages sent within teams
NOTION: weekly active teams
DROPBOX: files synced
LINKEDIN: monthly active members + engagement
NETFLIX: hours streamed
```

### 7.2 NSM characteristics

```
GOOD NSM
   ✓ Value-centric (customer-perceived)
   ✓ Predictive (leads revenue)
   ✓ Actionable (team can move it)
   ✓ Measurable (clear definition)
   ✓ Connected (multiple teams contribute)

BAD NSM (vanity)
   ✗ Volume-centric (visitors, signups, follows)
   ✗ Lagging (revenue itself — too late to act)
   ✗ Disconnected from product
   ✗ Multiple definitions (confusion)
```

### 7.3 NSM input metrics structure

```
NSM = WEEKLY ACTIVE TEAMS (Notion example)

INPUT METRICS (3-5 break down)

INPUT 1: NEW TEAM ACTIVATION RATE
   New teams reaching aha moment
   Owner: activation/onboarding team

INPUT 2: FEATURE ADOPTION BREADTH
   Teams using 3+ features
   Owner: core product team

INPUT 3: COLLABORATION DEPTH
   Avg members per team
   Owner: growth team

INPUT 4: RETENTION RATE
   Teams active month-over-month
   Owner: customer success

= each team owns 1 input
= alignment without micromanagement
```

### 7.4 PLG 5 foundations

```
1. NORTH STAR METRIC
   Aligned organization

2. TRANSPARENT PRICING
   Self-serve friendly
   Pricing page public

3. WELL-TIMED PAYWALLS
   Free → paid conversion natural
   Hit value moment first

4. BUILT-IN VIRALITY
   Product invites usage from new users
   Slack channels, Notion sharing, Figma multiplayer

5. FRICTIONLESS ONBOARDING
   Time-to-value (TTV) <10 min ideal
   Aha moment within first session
```

## 8. Activation + retention focus

### 8.1 Activation = predicador #1 retention

```
ACTIVATION EVENT

Definition: first "aha moment"
when user perceives core value

EXAMPLES
   Slack: send first message in team
   Notion: create first page
   Figma: collaborate with second user
   Dropbox: sync 1 file
   LinkedIn: connect with 5 people
   Spotify: listen 30 min (or save first song)

WHY ACTIVATION MATTERS
   Activated users retain 5-10x non-activated
   = activation event = single biggest retention lever
```

### 8.2 Activation rate optimization

```
ACTIVATION RATE
   = % new users reaching activation event
   Target: 20-40% activation rate healthy B2B SaaS
   Excellent: 50%+

OPTIMIZATION WORKFLOW
   1. DEFINE activation event (1-3 candidates)
   2. MEASURE current rate by cohort
   3. IDENTIFY drop-off points (funnel analysis)
   4. EXPERIMENT (A/B test onboarding flow)
   5. ITERATE based on data

8. **MQL→SQL 38% B2B SaaS** healthy benchmark.
9. **Win rates: 21% all / 29% qualified** B2B average.
10. **Pipeline coverage 3-5x** typical (varia 1.7x-6x by win rate).
11. **CAC payback 12-18 meses** B2B SaaS healthy.
12. **LTV:CAC 3:1+** sustainable.
13. **NRR 100%+** healthy; 110%+ outstanding; 130%+ best-in-class.
14. **Activation = predicator #1 retention** — 5-10x retention activated vs not.
15. **NSM** single value-centric measure + 3-5 input metrics.
16. **PLG 5 foundations**: NSM + transparent pricing + paywalls + virality + onboarding.
17. **2026 funnels = adaptive ecosystems** NÃO linear.

### Exemplo 4 — "Activation rate 12% — diagnostico + fix"

**Input:** "B2B SaaS PMF inicial. Activation rate 12% (way below 20-40% healthy). Diagnostico + fix?"

**Resposta esperada:**

> Sinal claro de friction onboarding ou activation event mal-definido. Plan 4-6 sem audit + optimization. **Razao**: 12% activation = 88% users signup mas nunca chegam aha moment = wasted CAC + leaky bucket retention impossible. Healthy 20-40% mid-market B2B SaaS, excellent 50%+. **Plan 4-6 sem**: **WEEK 1 Diagnostic**: (1) **Re-validate activation event definition** — eh measurable + value-centric + predictive of retention?; common mistake: definir activation muito early (signup = NÃO activation; precisa value perceived) OR muito late (paid conversion = retention not activation); ideal = "first valuable action completed"; (2) **Funnel analysis Mixpanel/Amplitude**: signup → step 1 onboarding → step 2 → … → activation event; identify biggest drop-off (usually step 3-5 of onboarding); (3) **Cohort retention curve** — activated users vs non-activated 30/60/90 day retention (typical 5-10x difference); (4) **Customer interviews 10-15** ("what was confusing during signup?", "did you reach value or quit before?"); (5) **Session recordings Hotjar/FullStory** — watch 20-30 onboarding sessions, identify rage-clicks + abandons; (6) **Heatmaps** signup form + onboarding pages; **WEEK 2 Hypothesize**: (7) Top 5 likely friction causes: (a) signup form too long (>5 fields = friction); (b) email verification required pre-onboarding (drop 30%); (c) onboarding skip-able with no smart defaults (users abandon mid-flow); (d) feature setup complex first session (too much commitment too early); (e) value moment too far away (>5 minutes = drop); (f) confusing terminology onboarding; (g) requiring credit card upfront free trial (kills conversion); (h) integrations required setup before value (chicken-and-egg); (i) no example data populated (blank slate intimidating); (j) no progress indicator (feels never-ending); **WEEK 3 Quick wins implementation**: (8) **Reduce signup friction** — name + email + password only (3 fields max); social login Google/Microsoft option (B2B); (9) **Email verification AFTER first value** (let users explore product first); (10) **Smart defaults onboarding** — pre-populated examples + sample data; (11) **Progressive disclosure** — show features as needed not all upfront; (12) **Time-to-value <5 minutes** — design onboarding to deliver value quickly; (13) **Skip-able optional steps** — required vs nice-to-have separated; (14) **Progress indicator** — "Step 2 of 4" reduces anxiety; (15) **In-app tooltips** strategic moments not all at once; (16) **Onboarding checklist** gamified (5 tasks → unlock value); (17) **Example workflow templates** — start FROM template not blank; **WEEK 4-5 A/B test + measure**: (18) Implement top 3-5 changes; (19) A/B test new vs old onboarding (50/50 split); (20) Measure 30 days minimum activation rate cohort; (21) Watch sessions new flow validate hypothesis; **WEEK 5-6 Iterate + scale**: (22) Iterate based data; (23) Document playbook; (24) Scale wins to all users; (25) Continue weekly cohort tracking; **Tactical examples activation events well-defined**: (a) **Slack** = "Send first message in workspace" (simple + value); (b) **Notion** = "Create first page" (low friction); (c) **Figma** = "Invite second collaborator" (multiplayer aha); (d) **Dropbox** = "Sync 1 file" (core utility); (e) **Calendly** = "Book first meeting via your link" (utility realized); para tu B2B SaaS workflow automation likely "successfully run first automated workflow" OR "connect 2+ integrations" (both signal value perceived); **Stats reference**: top quartile B2B SaaS 35-50% activation; (b) too many changes simultaneous (cant identify what worked); (c) skip A/B test (correlation vs causation); (d) ignore session recordings (data tells what, recordings tell WHY); (e) optimize for signup count vs activation (vanity); (f) require credit card upfront free trial (conversion killer typically); (g) email verification before exploration (kills 30%); (h) blank slate vs example template (chicken-and-egg); (i) too many onboarding steps mandatory (skip-able crucial);
