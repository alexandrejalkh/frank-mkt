## 4. Feynman Technique — explain como to a child

### 4.1 Origem + 4 steps

```
RICHARD FEYNMAN
   1965 Nobel Prize physicist
   Famous teacher + speaker
   "If you can't explain simply, you don't understand"

THE 4 STEPS

STEP 1: CHOOSE TOPIC
   Specific concept (NOT broad area)
   Vertical knowledge piece

STEP 2: EXPLORE + RESEARCH
   Deep dive
   Multiple sources
   Build understanding

STEP 3: EXPLAIN AS IF TO A CHILD
   Simple language (kindergarten level if needed)
   Analogies + examples
   Diagrams if helpful
   No jargon (or define inline)

STEP 4: TARGET GAPS
   Where struggled to explain simply?
   Re-research those areas
   Refine until simple
```

### 4.2 Aplicacao MKT posts

```
FEYNMAN PARA VERTICAL CONTENT MKT

EXAMPLE TOPICO COMPLEXO: "Reforma Tributaria BR Lei 122/2024"

❌ WITHOUT FEYNMAN
"A Reforma Tributaria sobre o Consumo trazida pela
Lei Complementar 122/2024 substitui ICMS, ISS, IPI,
PIS e COFINS pelo IBS e CBS, impondo aliquotas
combinadas e mecanismo de creditamento amplo..."
= jargon-heavy, lay audience tunes out

✅ WITH FEYNMAN
"Imagina que voce e dono de loja de roupas. Hoje voce
paga 5 impostos diferentes (ICMS, ISS, IPI, PIS, COFINS)
quando vende. E uma confusao. A Reforma Tributaria troca
esses 5 por apenas 2 (IBS e CBS), que se somam numa
conta unica.

Mais simples? Sim. MAS o detalhe importante: a
aliquota total NAO muda muito (cerca de 26-27%).
O que muda e como voce calcula + quando paga +
para QUEM paga (estado vs federal).

Para quem isso importa?
- Voce que vende produtos/servicos (precisa adaptar)
- Voce contador (precisa atualizar sistemas)
- Voce empresario que quer entender seu custo real

3 pontos pra acompanhar:
1. Vigencia gradual ate 2033 (NAO e overnight)
2. Cashback para baixa renda (novo, conferir lista produtos)
3. Sistema split-payment (pagamento separado fornecedor + governo)"

= simple language, analogies, structure, audience-relevant
```

### 4.3 Sinal que understanding e raso

```
RED FLAGS quando applying Feynman

❌ Cant explain without using technical terms
   = need re-research

❌ Audience asks "what does X mean?" frequently
   = simplification incomplete

❌ Analogies feel forced
   = trying explain something not understood

❌ Long sentences with multiple clauses
   = thinking unclear

GREEN FLAGS

✅ Lay person explains back accurately
✅ Children can follow basic concept
✅ Sentences short + active voice
✅ Visual diagram possible 1-page
✅ Analogy from common life domain
```

### 4.4 Tools + workflow Feynman para posts

```
WORKFLOW FEYNMAN PRODUCING VERTICAL POST

STEP 1 (30 min): Choose topic + outline
   1 specific concept
   3-5 sub-points max

STEP 2 (60-120 min): Research
   3-5 sources high quality
   Take notes own words

STEP 3 (30-60 min): First draft
   Simple language
   Analogies
   Examples

STEP 4 (30 min): Lay person test
   Ask non-expert read
   Note confusion points

STEP 5 (30 min): Refine
   Simplify confusion points
   Replace jargon
   Add visuals if helpful

= 3-5 hours per post
= high quality + simple + authoritative
```

## 5. Tom adaptation por audience

### 5.1 3 tom levels per vertical

```
TECNICO LEVEL (peer-to-peer specialist)

AUDIENCE
   Other engineers/lawyers/CFOs
   Same profession or adjacent
   Want depth + nuance

LANGUAGE
   Industry jargon OK (assumed knowledge)
   Code samples (TI)
   Specific frameworks (Negocios)
   Case law (Juridico)
   Specific products/tools

DEPTH
   High
   Long-form OK (3000+ palavras)
   Technical detail welcome

EXEMPLOS
   GitHub README (cf. `github-presence`)
   Engineering blog post
   Legal whitepaper
   CFO Magazine article

EXECUTIVE LEVEL (C-level + decision-makers)

AUDIENCE
   Founders, C-level, board members
   Strategic decisions
   Want frameworks + ROI

LANGUAGE
   Business jargon OK
   Less technical detail
   Strategic implications focus

DEPTH
   Medium
   1500-3000 palavras typical
   Frameworks + data + examples

EXEMPLOS
   LinkedIn long-form (cf. `linkedin-organico`)
   Harvard Business Review-style
   Boardroom presentations
   Industry reports

LEIGO LEVEL (educational general)

AUDIENCE
   Curious general public
   Beginners + adjacent fields
   Want understanding + relevance

LANGUAGE
   Plain language
   Analogies (Feynman Technique)
   Define jargon inline
   Storytelling

DEPTH
   Lower
   500-1500 palavras typical
   Examples + visuals

EXEMPLOS
   Newsletter educational
   Twitter/X thread accessible
   YouTube explainer
   TikTok educational
```

## 8. Team-led content strategy

### 8.1 Team-led vs Brand-led

```
BRAND-LED (legacy)
   Corporate page LinkedIn primary
   Marketing team writes
   Brand voice singular
   Coverage limited (team size)
   Algorithm distribution narrow

TEAM-LED (2026 winning)
   Personal LinkedIn employees primary
   SMEs + founders + experts write
   Brand voice = chorus of voices
   Coverage broad (network of experts)
   Algorithm distribution wider

WHY TEAM-LED WINS 2026
   LinkedIn algorithm favors personal accounts
   SME authority de-risks B2B buying
   Authentic voices > polished corporate
   Compounds over years
```

### 8.2 Implementation framework

```
TEAM-LED CONTENT IMPLEMENTATION

STEP 1: IDENTIFY SMEs
   - Founder/CEO (vertical authority)
   - Engineering leaders (TI vertical)
   - Compliance officers (Juridico)
   - CFO (Financeiro/Fiscal)
   - HR leader (RH)
   - Operations head (ADM)
   - 5-15 SMEs typical
```
